📋 What is Kantier? READ FIRST

Kantier is Albania's first construction site management SaaS platform, built by Aurora Baba (founder, architect, UCL London). It is modelled after Fieldwire but designed for Albanian construction teams. Aurora has no coding background — always explain before doing, never run commands without explaining what they do.

Live URL: kantier.vercel.app | Stack: Next.js 16.2 + Supabase + Vercel | GitHub: github.com/aurorababa/kantier

Working method: Claude delivers complete files for download. Aurora applies them: Cmd+A → Delete → Paste → Cmd+S in VS Code. Deploy: git add . && git commit -m "..." && git push from ~/kantier.

This document URL: https://kantier-ui-reference-document.vercel.app

🧠 Key Design Decisions Made

  • Light theme everywhere — white nav (#FFFFFF), light grey background (#F4F5F7), white panels. Matches Fieldwire's professional construction feel.
  • Teardrop pins with Albanian category acronyms — EL=Elektrike, HI=Hidraulike, MU=Muraturë, DR=Dritare & Dyer, etc. Color = priority level.
  • Priority system — Prioritet 1 (red), P2 (orange), P3 (yellow), Verifikuar (blue), Mbaruar (green), Ulët (gray). Old values urgjent/larte/mesem still supported for backward compatibility.
  • Grayscale PDF thumbnail with colored teardrop pin on top — light gray range (80–245), pin stays full color. Same in both Planet and Detyrat views.
  • Unified task detail — same fields in both Detyrat sidebar and Planet right panel: Status, Priority, Category, Assigned to, Start date, Deadline, Completion date, Cost, Manpower, Description, Checklist, Photos, Comments, Plan thumbnail.
  • Scope creep tracking — data_afatit (planned deadline) vs data_mbarimit (actual completion). Delta = scope creep. Will be used in Reports.

✅ What is Working Right Now (3 April 2026)

  • Dashboard — light theme, stat cards, project list with progress bars, recent tasks
  • Detyrat (project tasks) — donut chart, filter tabs, task list with priority bars + category acronym badges, full task detail panel
  • Task detail panel — all fields save to Supabase on blur, photos upload to foto-kantier bucket, checklist, comments/activity log
  • Planet (plan viewer) — PDF upload, zoom/pan/scroll, teardrop pins with category acronyms, priority colors, pin label tooltip on select
  • Clicking a pin → full task detail in right panel (same fields as Detyrat)
  • Plan thumbnail in task detail — grayscale PDF with colored teardrop pin at exact location
  • Clicking thumbnail in Detyrat → navigates to Planet and auto-opens correct plan
  • + Existing task — search modal when placing a pin to link an existing task
  • Priority dropdowns updated in both files
  • Category list unified: EL, HI, BE, SU, LY, MU, FA, PL, ME, DR, CA, IZ, GI, HV, AR, CI, ST, TJ

🎯 Fieldwire features still missing from Kantier

  • Auto-activity log — Fieldwire automatically records every change: "Changed start date to 3/3/26", "Changed priority to P2", "Aurora Baba uploaded photo". Kantier only has manual comments. Need to add auto-logging in saveTaskField.
  • Assignee dropdown — Fieldwire shows project members in a dropdown. Kantier has a free-text email field. Need to replace with dropdown from anetaret table.
  • Related tasks — Fieldwire shows linked tasks with + New task / + Existing task. Partially built (existing task search modal) but not linked to a related_task_ids field.
  • Photo metadata — Fieldwire shows who uploaded a photo and when. Kantier stores only the URL. Need to store {url, email, timestamp} objects instead of plain strings.
  • Link task immediately on creation — when creating a new task from Detyrat, offer to immediately link it to a plan location.

📸 Photo metadata migration plan

Currently foto_urls stores plain strings: ["https://...jpg", "https://...jpg"]

Needs to become: [{"url": "https://...", "email": "aurora@...", "data": "2026-04-03T10:22:00Z"}, ...]

Migration approach: In Supabase SQL editor: ALTER TABLE detyrat ALTER COLUMN foto_urls TYPE jsonb USING foto_urls::jsonb; — this preserves existing data. Then update uploadPhotoToTask() in both planet/page.tsx and projektet/[id]/page.tsx to store objects instead of strings. The photo display component needs to read f.url instead of f directly.

Raporte Page — Plan
What the reports page should contain and how to build it.

📊 What Reports Need to Show

The reports page lives at /dashboard/projektet/[id]/raporte. It reads from the detyrat table for the current project and calculates these metrics:

  • Tasks by status — bar or donut: Hapur / Në Progres / Mbaruar. Count and % of total.
  • Overdue tasks — tasks where data_afatit < today AND statusi ≠ mbaruar. List with task name, deadline, how many days overdue.
  • Scope creep per task — tasks where data_mbarimit > data_afatit. Shows planned vs actual date, delay in days. This is the most important construction metric.
  • Cost by category — bar chart: EL / HI / MU etc. with total cost (€) per category.
  • Manpower hours by category — same grouping as cost but shows njerezit_punojne values.
  • Tasks per assignee — who has the most tasks, and what % are complete.
  • Photo log — after foto_urls migration: chronological list of all task photos with uploader, date, task name, category.

🔧 Technical approach for Raporte

Use a single Supabase query: supabase.from('detyrat').select('*').eq('projekti_id', id) then do all calculations in JavaScript — no complex SQL needed.

Charts: Use inline SVG (no library needed). Simple bar charts can be built with rect elements sized to percentage values. The donut chart pattern is already in projektet/[id]/page.tsx — reuse it.

Export: Add a "Shkarko PDF" button that calls window.print() with print-specific CSS (@media print). This lets the site manager share reports with clients without needing a PDF library.

Build order: (1) Status breakdown + overdue list first. (2) Scope creep table. (3) Cost/manpower charts. (4) Photo log after the foto_urls migration.

Foto Page — Plan
The Foto tab currently links to /foto but has no content. Here's what to build.

📸 What the Foto Page Should Be

The Foto page at /dashboard/projektet/[id]/foto should be a visual construction diary — every photo taken on site, organized chronologically and by category.

  • Photo grid — masonry or uniform grid of all task photos for this project. Each shows: thumbnail, task name, category acronym badge, date, uploader initials.
  • Filter by category — EL / HI / MU tabs at top to filter photos by work type.
  • Filter by date range — date picker to see photos from a specific period.
  • Lightbox — click a photo to see full size with all metadata: task name, who took it, when, description.
  • Link back to task — each photo card has a "→ Shiko detyrën" button that opens the task detail.
  • Standalone upload — upload photos not linked to a specific task, just to the project's site diary.

Prerequisite: The foto_urls migration (see feature #4) must be done first so photos have uploader and timestamp metadata.

Fieldwire Reference
Key Fieldwire patterns that Kantier is modelling. Reference for design decisions.
Task attributes panel (right side)
Status (Priority dropdown) · Category · Assignee · Plan thumbnail with pin · Location text · Start date · End date · Manpower (hours) · Cost (USD) · Tags · Watchers

Key difference from Kantier: Fieldwire's "Status" IS the priority (P1/P2/P3/Completed/Verified). Kantier separates these into Statusi and Prioriteti which gives more granularity.
Task main panel (left side in Fieldwire)
Title (editable) · Related Tasks (+ New / + Existing) · Checklist (+ items, + full checklist) · Activity feed (auto-log of ALL changes + manual comments + @mentions)

Kantier status: Has title, checklist, comments. Missing: related tasks column, auto-activity log, @mentions.
Plan thumbnail (Fieldwire)
Grayscale PDF rendering of the plan page. Full color teardrop pin at the task location. Plan name + sheet number below. Clicking opens the plan viewer focused on that task's pin.

Kantier status: ✓ Working — grayscale PDF, colored pin, plan name, click navigation.
Pin design (Fieldwire)
Teardrop shape. Initials of assignee OR category abbreviation. Priority color fill. On hover: tooltip with task title + task number (#12). Right panel shows numbered task list.

Kantier status: ✓ Teardrop shape. ✓ Category acronym. ✓ Priority colors. ✓ Label on select. ✗ Assignee initials option missing.
