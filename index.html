Platform Overview
What Kantier is, what's built, and the database structure.

✅ Currently Built & Deployed

Dashboard — Light theme (Fieldwire-style). Stat cards (projects, tasks open/in-progress/done), project list with progress bars, recent tasks panel, status breakdown chart. Sidebar with project navigation.

Project View (/projektet/[id]) — Donut chart showing task completion %. Filter tabs (all/open/in-progress/done). Task list with priority color bars, status/category badges, overdue date highlighting. Full task detail panel opens on right when task selected.

Task Detail Panel — Title edit, Status, Priority, Category, Assigned to, Start date, Deadline, Completion date (data_mbarimit), Cost, Manpower, Description, Checklist, Photos, Comments/Activity log. Plan link button if task has a pin.

Plan Viewer (/planet) — PDF upload, zoom/pan, pin placement, pin click opens full task detail in right panel. Plans organized by category grid.

PDF Zoom Fix — Render queue prevents canvas race condition. PDF loads once, re-renders only on scale/page change.

🔧 Known Issues (as of Session 1 end)

1. Plan thumbnail not showing in task detail — The "Vendndodhja në plan" card shows in projekti/page.tsx but only navigates to the plan list, it does not show an actual thumbnail image of the plan with the pin location highlighted.

2. Pin click label — When hovering or clicking a pin in the plan viewer, the task title should appear as a tooltip/label next to the pin (like Fieldwire). Currently no label shows.

3. Pin click → full task modal — In the plan viewer, clicking a pin opens the detail in the right panel, but it should open the SAME full task modal as in Detyrat view (not just the side panel). These two views need to be unified.

4. More priority levels — Fieldwire uses Priority 1, 2, 3, Completed, Verified. Kantier currently has Urgjent/Larte/Mesem/Ulet. Need to add more levels and match colors to pin colors.

5. Related tasks — Ability to link tasks to each other (like Fieldwire's "Related Tasks" with + New task / + Existing task).

6. @ mentions in comments — Fieldwire allows @plan and @person mentions in comments with autocomplete dropdown.

Plan Viewer — Specific Tasks
Detailed breakdown of what needs to change in planet/page.tsx

🗺 Plan Viewer Current State

The plan viewer works well for basic use. PDF loads, zoom works (fixed in session 1), pins can be placed, clicking a pin in select mode opens a detail panel on the right. The right panel shows all task fields, checklist, photos, comments.

What's missing compared to Fieldwire:

→ Pin labels: Fieldwire shows a tooltip with the task title when you hover or select a pin. In drawOverlay() we draw circles but no text next to them.

→ Pin number: Fieldwire numbers each pin (#1, #2...). Our pins show "!" which is not informative.

→ Pin colors by priority: Fieldwire uses red/orange/yellow/green/blue exactly matching the priority level. Our colors are close but the priority system needs expanding.

→ Markup tools: Fieldwire has drawing tools (pen, arrow, text, shapes) for annotating PDFs. This is a larger future feature.

🔧 How to fix pin labels — step by step

Open src/app/dashboard/projektet/[id]/planet/page.tsx and find the drawOverlay() function (around line 110). After the circle is drawn for each pin, add this code to draw the task title as a label:

The logic: if a pin is selected (isSelected), draw a white rounded rectangle next to the pin, then draw the task title text inside it. This is pure Canvas 2D API — no extra libraries needed.

Code to add after ctx.stroke() inside the forEach:

if (isSelected) {'{'} const label = p.titulli.substring(0,25); const lw = ctx.measureText(label).width + 16; ctx.fillStyle = 'white'; ctx.beginPath(); ctx.roundRect(px+18, py-12, lw, 24, 4); ctx.fill(); ctx.fillStyle = '#1C2333'; ctx.font = 'bold 11px sans-serif'; ctx.fillText(label, px+26, py+4); {'}'}

Fieldwire — Task Detail Panel
Reference for what the task detail should contain and look like.

📋 Fieldwire Task Panel Structure

Left side (main content): Title (editable), Related Tasks section (+ New task / + Existing task search), Checklist (+ New item / + Add checklist), Activity feed (comments + change log with timestamps and @mentions).

Right side (attributes): Status (Priority 1/2/3/Completed/Verified + Delete), Category (dropdown), Assignee (person picker), Plan thumbnail (shows miniature of the plan with pin location), Location (text), Start date, End date, Manpower (hours), Cost (USD), Tags, Watchers.

Key differences from Kantier current state: (1) Plan thumbnail is a real rendered image with the pin dot. (2) Status uses Priority 1/2/3 not text labels. (3) Activity log shows all field changes automatically, not just comments. (4) @mention autocomplete in comments.

Fieldwire — Plan Viewer & Pins
How Fieldwire handles the plan viewer — reference for future improvements.

🗺 Fieldwire Plan Viewer Features

Toolbar (left side): Full screen, zoom in/out, pan, select, pin/task, markup tools (pen, line, arrow, rectangle, text, stamp), undo. Dark slim toolbar matching Kantier's current approach.

Pin design: Colored circle with initials of assignee (e.g. "DR", "PL"). Priority color background. Shows task number (#12). On hover: tooltip with task title.

Task list (right panel): Shows all tasks on current plan. Each row shows: assignee avatar, task number, plan ref, title. Clicking opens the task detail modal/overlay.

Version control: Bottom bar shows plan name and version date. Can switch between versions.

Attachment pins: Separate pin type for attaching PDFs (RFIs, specs) to a location on the plan. Shows a paperclip icon, clicking opens the attachment.
