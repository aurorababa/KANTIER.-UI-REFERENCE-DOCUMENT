📋 What is Kantier? READ FIRST

Kantier is Albania's first construction site management SaaS platform, built by Aurora Baba (founder, architect, UCL London). It is modelled after Fieldwire and PlanRadar but designed for Albanian construction teams. Aurora has no coding background — always explain before doing, never run commands without explaining what they do.

Live URL: kantier.vercel.app | Stack: Next.js 16.2 + Supabase + Vercel | GitHub: github.com/aurorababa/kantier

Working method: Claude delivers complete files for download. Aurora applies them: Cmd+A → Delete → Paste → Cmd+S in VS Code. Deploy: git add . && git commit -m "..." && git push from ~/kantier.

As PMP advisor: Claude advises as an experienced PMP-accredited construction project manager familiar with Procore, Fieldwire, and PlanRadar. Kantier is being built following PMBOK and ISO 9001 standards where applicable.

✅ What is Working Right Now (3 April 2026)

Dashboard: Light theme, stat cards, project list with progress bars, recent tasks, sidebar navigation.

Detyrat (tasks): Donut chart, filter tabs, task list with priority bars + category acronym badges, full task detail panel. Status/Priority dropdowns are clean flat style with colored dot. data_mbarimit removed per decision.

Auto-Activity Log: Every change to a task is automatically logged in komentet with type:'activity'. Logs: field changes (status, priority, category, dates, cost, manpower, title, description, assignee), photo uploads/deletes, checklist add/toggle/delete. Displayed as italic gray lines with pencil icon. Manual comments remain as chat bubbles.

Planet (plan viewer): PDF upload, zoom/pan/scroll, teardrop pins with category acronyms, priority colors, pin label tooltip on select, task detail right panel.

Raporte: Date filter (week/biweekly/month/custom), Summary + Detailed PDF modes, ISO 9001 cover page with document control block, auto-numbering (RPT-YYYY-CODE-NNN), Draft/Issued status, R00-R09 revisions, document registered in `dokumentet` table on print.

Detaje: Full project details page — kodi_projektit, klienti, kompania, menaxher_projekti, data_fillimit_projektit, data_mbarimit_projektit, statusi_projektit, vlera_kontrates, siperfaqja, faza. Kartela card preview. Data feeds into Raporte cover page automatically.

⚠ Known Decisions Made

data_mbarimit removed from sidebar: By agreement, the "Data e Mbarimit" (actual completion date) field has been removed from the task detail sidebar. The field still exists in Supabase and the activity log still references it in labels — but it is no longer shown to users. Scope creep calculation in reports is deprioritized for now.

Revision R00-R09: Extended from R00-R04 to R00-R09 to cover longer projects. Each revision = same document re-issued with corrections. New document = new number.

data_mbarimit P1 bug skipped: Decided not to fix the save reliability bug for data_mbarimit since the field was removed from the UI.

Active Bugs
Known issues as of 3 April 2026.
PriorityBugRoot CauseFix Approach
P1 Date fields (data_fillimit, data_afatit) not logging in activity log
Dates save to Supabase correctly but don't trigger activity log entry.		 The norm() comparison function converts null → '' and the date value → first 10 chars. When the current value is null and user enters a date, norm(null)='' vs norm('2026-04-03')='2026-04-03' — these ARE different so it should log. Root cause may be onBlur firing before the date picker commits the value, or the value being the full ISO string from Supabase. In page.tsx saveTaskField: add console.log to trace what current and normalized actually are when a date field blurs. The fix is likely in how the date input passes value on blur vs onChange.
P2 caktuar_email is free-text input
Should be a dropdown from anetaret table.		 Not yet built. anetaret table exists but not queried. In loadData(), fetch anetaret. Replace text input with select. Fallback to text if no members.
P2 foto_urls missing metadata
Photos stored as plain URL strings, no uploader or timestamp.		 foto_urls is string[] not object[]. SQL migration + update uploadPhotoToTask. Display shows f.url instead of f directly.
P3 planet/page.tsx activity log not updated
planet/page.tsx has its own copy of saveTaskField without the new saveWithLog() pattern.		 The S3 updates were only applied to projektet/[id]/page.tsx, not planet/page.tsx. Apply the same saveWithLog() pattern to planet/page.tsx. Requires uploading that file in a future session.

Strategy & Roadmap
Agreed priorities as of 3 April 2026. Discuss with Aurora before starting each step.

🎯 Strategic Direction PMP Advised

Kantier has completed its core MVP foundation in Session 3. The platform now has: task management, plan viewing, photo documentation, a professional ISO 9001-compliant reporting system, and project details tracking. This is a strong foundation.

The next phase focuses on data quality and usability — ensuring the data that's already being collected is clean, correctly attributed, and accessible. This means: assignee dropdown (so tasks are properly owned), photo metadata (so the Foto page can be built), activity log for planet view, and the global task list.

After that, new construction management modules — Document Register page, Foto page, and eventually RFI and Change Order workflows.

Design Decisions
Key decisions made — do not reverse without discussion.
DecisionRationale
Light theme everywhere — white nav (#FFFFFF), #F4F5F7 background, white panelsMatches Fieldwire's professional construction feel. Fieldwire uses pure white+gray — no dark mode.
Teardrop pins with Albanian category acronyms — EL, HI, MU etc. Color = priorityImmediately identifies work type on crowded plans. Color shows urgency at a glance.
Flat status/priority dropdowns — white bg, colored dot only, no emoji, no colored bg fillsCleaner for professional use. Emojis (🔴🟠🟡) removed as they look unprofessional in printed reports.
Neutral report palette — #1C2333 headings, #5A6474 secondary, ruled lines not colored blocksReports are shared with clients. Dense colored cards are inappropriate for formal construction documents.
komentet stores both comments and activity log — distinguished by type:'activity'No new table needed. Same feed shows both. Activity entries are italic gray; comments are bubbles.
Document numbering: TYPE-YEAR-CODE-NNN — RPT-2026-PT01-001PMBOK standard. Each document type has its own series. Revision = same doc re-issued. New doc = new number.
data_mbarimit hidden from UI — field exists in DB but not shownP1 bug with save reliability + scope creep feature deprioritized. Field preserved for future use.
Kantier separates Statusi and PrioritetiFieldwire's "Status" IS the priority. Kantier separates them for more granularity — allows "Hapur + P1" vs "Hapur + P3".
