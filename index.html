📋 What is Kantier? READ FIRST

Kantier is Albania's first construction site management SaaS, built by Aurora Baba (architect, UCL London, no coding background). Modelled on Fieldwire + PlanRadar. Stack: Next.js 16.2 + Supabase + Vercel. Live: kantier.vercel.app. GitHub: github.com/aurorababa/kantier.

Working method: Claude delivers complete files. Aurora applies Cmd+A → Delete → Paste → Cmd+S. Deploy: cd ~/kantier && git add . && git commit -m "..." && git push

Claude acts as PMP-accredited construction PM familiar with Procore, Fieldwire, PlanRadar. PMBOK and ISO 9001 standards apply.

Active test project: Net Zero Energy Residential House (USA, wood-frame, 93 drawings). Aurora is uploading individual plan sheets to Planet. 34 tasks defined — see section 09.

12
Design Decisions
Locked — do not reverse without discussion
DecisionRationale
Light theme everywhere — white nav, #F4F5F7 bgMatches Fieldwire. No dark mode.
Teardrop pins + category acronyms — color = priorityType + urgency readable at a glance on crowded plans.
Flat dropdowns — white bg, colored dot, no emoji, no fillsProfessional. Emojis render badly in printed PDFs.
Neutral report palette — ruled lines, not colored blocksReports go to clients and authorities. Dense colors are informal.
komentet stores both types — type:'activity' differentiatesNo extra table. Activity = italic gray + pencil. Comment = bubble.
TYPE-YEAR-CODE-NNN numberingPMBOK standard. Each type has own series. Revision ≠ new number.
data_mbarimit hiddenSave bug + scope creep deprioritized. Field kept in DB.
Statusi and Prioriteti are separateMore granularity than Fieldwire — "Hapur + P1" vs "Hapur + P3".
PDF only, no Word exportWord is editable and defeats ISO 9001 document control.
13
Document System — Design
Layout and fields for each of the 5 document types

📄 Shared ISO 9001 header All types

Every document type shares the same control block at the top: Document Number, Revision, Status (Draft/Issued), Prepared by, Approved by, Date Issued. Auto-generated from dokumentet table. Cannot be edited after issuance.

The body layout below the control block differs per type. Cross-references stored as references jsonb[] on dokumentet (field to be added): e.g. ["RFI-2026-PT01-002", "A-105"]. Appear in document footer.

TypeBody structureKey fieldsReferences
RPT Site ReportNarrative — project stats + task list + embedded photos per task + overdue items + cost summaryReporting period, tasks included, status breakdown, total costTask IDs, open RFI numbers
RFI Request for InfoQuestion form — drawing reference + free-text description box + response field + response dateDrawing number, grid ref, question text, responder, response, date requiredDrawing sheet code (e.g. A-105 Grid C-3), related task ID
INS Inspection ReportChecklist — pass/fail per item + notes + inspector + AHJ sign-off blockInspection type, phase, inspector, date, pass/fail per itemPlan sheet, phase, task IDs inspected
COR Change Order RequestFinancial form — reason + scope + cost table (original vs new) + schedule impact + approval blockOriginal value, change amount, new total, schedule impact (days), approval statusRFI that triggered change, affected task IDs
DCO Design Change OrderDesign change log — original spec + revised spec + affected drawing list + engineer sign-offDiscipline, original drawing ref, revised ref, reason, engineer nameOriginal drawing, related COR or RFI
