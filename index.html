Instructions for Claude — READ FIRST

Working method: Claude delivers complete files named with the convention below. Aurora applies Cmd+A → Delete → Paste → Cmd+S in the correct VS Code tab. Always verify breadcrumb path before pasting. Deploy: cd ~/kantier && git add . && git commit -m "..." && git push

What is Kantier?

Kantier is Albania's first construction site management SaaS, built by Aurora Baba (architect, UCL London, no coding background). Stack: Next.js 16.2 + Supabase + Vercel. Claude acts as a PMP-accredited construction PM familiar with Procore, Fieldwire, PlanRadar. PMBOK and ISO 9001 standards apply to all document decisions.

Critical Rules

  • Never call loadData() after field saves — causes race condition where task dates snap back to old DB values. Update local state directly with setSelectedTask + setDetyrat.
  • Date inputs must save on onChange, not onBlur — onBlur races with loadData() re-fetch.
  • File naming convention is mandatory — see Section 09. All files delivered must use route_path_page.tsx format.
  • params.id must be typed — always use const projektId = params.id as string after const params = useParams(). Never let sed replace params.id inside its own declaration.
  • No sed commands on TypeScript files — use Python string replace or direct str_replace instead.
  • PDF only, no Word export — Word is editable and defeats ISO 9001 document control.
  • data_mbarimit hidden from UI — field preserved in DB but never shown.

Active Test Project

Net Zero Energy Residential House (USA, wood-frame). Project_Test03 on Kantier. See Section 12 for full task list.

Strategy & Roadmap

Where we are: Phase 1 (bug fixes) is complete. All 5 document types work. Document register exists. Document detail page exists but needs routing validation and testing. Phase 2 (Master Log) is next.

Phase 1 — Bug Fixes ✅ Complete

  • Date saving bug fixed (onChange, no loadData race)
  • RPT date filter fixed (active-during-period logic)
  • Dokumentet tab in all navs
  • Doc number links to detail page

Phase 2 — Master Log (next session)

The current /dokumentet page is a good register but lacks the live document management workflow that a PM needs daily. The goal is a side-panel pattern (same as Detyrat task detail) so you never leave the register to manage a document.

  • The detail page at /dokumentet/[docId] works but is only reachable by clicking the underlined doc number
  • Better pattern: clicking any row in the register opens a right side panel (like Detyrat) without navigating away
  • The panel shows: workflow stepper, form content, response upload, activity log, "Dërgo" button
  • This eliminates the need for a separate page and matches the Fieldwire/Procore UX pattern

Phase 3 — Task Linking

  • Add dokument_id reference to detyrat table
  • Task detail shows linked RFIs/INS documents
  • Document detail shows the task it came from

Phase 4 — Notifications (future)

  • "Dërgo" button emails PDF to responsible person
  • Supabase Edge Functions + Resend
  • Separate sprint

Working with Claude

Start each session

Paste URL: https://kantier-ui-reference-document.vercel.app. Claude fetches and has full context. State which feature to work on and which phase.

Upload files

Drag files from VS Code Explorer into Claude chat one at a time. When uploading multiple files, wait for Claude to confirm the line count before uploading the next. All files share the name page.tsx — Claude identifies them by line count. Always state which route the file belongs to.

Click correct file tab in VS Code (verify breadcrumb path at top). Cmd+A → Delete → Cmd+V → Cmd+S. Check PROBLEMS = 0. Then deploy.

⚠ Common Mistakes

  • Pasting into wrong VS Code tab — always check the breadcrumb path at top of editor
  • Multiple files uploaded at once — only the last one comes through. Upload one at a time.
  • params.id typing — always const params = useParams() then const projektId = params.id as string on the next line
  • sed on TypeScript — never use sed on .tsx files. Use Python string replace or str_replace tool.
  • Wrong terminal — must be ~/kantier not ~/kantier-website
  • Reference doc update — update on GitHub after each session so Vercel deploys the new version

Design Decisions — Locked

Do not reverse without discussion.

DecisionRationale
Light theme everywhere — white nav, #F4F5F7 bgMatches Fieldwire. No dark mode.
Teardrop pins + category acronyms — color = priorityType + urgency readable at a glance on crowded plans.
Flat dropdowns — white bg, colored dot, no emoji, no fillsProfessional. Emojis render badly in printed PDFs.
Neutral report palette — ruled lines, not colored blocksReports go to clients and authorities. Dense colors are informal.
komentet stores both activity + commentsNo extra table. Activity = italic gray + pencil. Comment = bubble.
TYPE-YEAR-CODE-NNN numberingPMBOK standard. Each type has own series. Revision ≠ new number.
data_mbarimit hidden from UISave bug + scope creep deprioritized. Field kept in DB.
Statusi and Prioriteti are separateMore granularity than Fieldwire — "Hapur + P1" vs "Hapur + P3".
PDF only, no Word exportWord is editable and defeats ISO 9001 document control.
Revisions R00–R05 onlyReduced from R09. RPT locked at R00 (no revision).
Date inputs save on onChangeonBlur causes race condition with loadData(). onChange is immediate and reliable.
No loadData() after field savesUpdates local state only. loadData() overwrites selectedTask and causes dates to snap back.

Document System

Shared ISO 9001 header — all types

Every document type shares the same control block: Document Number, Type, Revision, Status (Draft/Issued), Prepared by, Approved by, Date Issued. Auto-generated from dokumentet table. Cannot be edited after issuance.

Document body per type

TypeBody structureEditable untilKey action
RPTAuto-generated from tasks — stats, overdue, cost, assignee table, activity log per taskNever (read-only)—
RFIDrawing ref + grid ref + question (blue panel) + drawing screenshot + site photo + response block (green when filled)Response editable until closedKrijo COR button when answered
INSInspection type + phase + inspector + checklist (pass/fail/NA per item) + result summary + AHJ sign-offResults editable until passed/conditional—
CORReason + scope + line-item cost table + financial summary (original→delta→new) + schedule impact + approval statusApproval status editable until approved/rejected—
DCOBefore/after panels (red/green) — original drawing + spec + screenshot vs revised drawing + spec + screenshot + reason + affected docsUntil issued—

Document numbering

Format: TYPE-YEAR-CODE-NNN e.g. RFI-2026-PT01-001

Code = kodi_projektit from projektet table (max 8 chars, alphanumeric). Each type has its own sequence. Revisions R00–R05. Re-issuing same document = new revision. New document = new number.
