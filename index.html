TURBOPACK RULE — Never Break
Never use template literals (`${var}`) directly inside JSX children of <option> tags. Use array join instead: {[a.profiles?.emri, a.profiles?.mbiemri].filter(Boolean).join(' ') || a.email?.split('@')[0]} and role: {a.roli ? ' (' + a.roli + ')' : ''}

What is Kantier?

Albania's first construction site management SaaS built by Aurora Baba (architect, UCL London, no coding background). Stack: Next.js 16.2 + Supabase + Vercel. Claude acts as a PMP-accredited construction PM familiar with Procore, Fieldwire, PlanRadar.

Critical Rules

RuleReason
Never call loadData() after field savesRace condition — dates snap back. Update local state with setSelectedTask + setDetyrat instead.
Date inputs save on onChange, not onBluronBlur races with loadData() re-fetch.
File naming convention is mandatorySee Section 04. Route-based name prevents wrong-tab mistakes.
params.id must be typedconst projektId = params.id as string. Never inline.
No sed on TypeScript filessed breaks on special chars. Use str_replace tool only.
Upload files one at a timeMultiple uploads overwrite each other. Wait for line count confirmation.
PDF only, no Word exportWord is editable — defeats ISO 9001 document control.
No template literals in JSX option textTurbopack build fails. Use [a, b].filter(Boolean).join(' ')
foto_urls stores objects, not plain stringsParse with: typeof v==='string' && v.startsWith('{') ? JSON.parse(v).url : (v?.url||v)
Trial logic is inlined in each fileDo NOT import from @/lib/trial — import caused resolution issues in this Turbopack version.
confirm() is blocked on VercelUse inline state-based confirmation UI instead of browser confirm() dialog.
RLS policies use user_id = auth.uid()Email-based RLS fails silently. Always match on user_id. Trigger auto-fills user_id on anetaret insert.
profiles RLS allows all authenticated readsCREATE POLICY "profiles_read_all" ON profiles FOR SELECT TO authenticated USING (true);

SQL Applied in S11

-- RLS: members can read projects they belong to DROP POLICY IF EXISTS "projektet_member_read" ON projektet; CREATE POLICY "projektet_member_read" ON projektet FOR SELECT TO authenticated USING ( krijuar_nga = auth.uid() OR id IN (SELECT projekti_id FROM anetaret WHERE user_id = auth.uid()) ); -- RLS: members can read tasks in their projects DROP POLICY IF EXISTS "detyrat_member_read" ON detyrat; CREATE POLICY "detyrat_member_read" ON detyrat FOR SELECT TO authenticated USING ( projekti_id IN (SELECT id FROM projektet WHERE krijuar_nga = auth.uid()) OR projekti_id IN (SELECT projekti_id FROM anetaret WHERE user_id = auth.uid()) ); -- RLS: members can update and insert tasks CREATE POLICY "detyrat_member_update" ON detyrat FOR UPDATE TO authenticated USING (projekti_id IN (SELECT id FROM projektet WHERE krijuar_nga = auth.uid()) OR projekti_id IN (SELECT projekti_id FROM anetaret WHERE user_id = auth.uid())); CREATE POLICY "detyrat_member_insert" ON detyrat FOR INSERT TO authenticated WITH CHECK (projekti_id IN (SELECT id FROM projektet WHERE krijuar_nga = auth.uid()) OR projekti_id IN (SELECT projekti_id FROM anetaret WHERE user_id = auth.uid())); -- RLS: owners can delete members CREATE POLICY "anetaret_delete" ON anetaret FOR DELETE TO authenticated USING (projekti_id IN (SELECT id FROM projektet WHERE krijuar_nga = auth.uid())); -- RLS: all authenticated users can read profiles (needed for name display) CREATE POLICY "profiles_read_all" ON profiles FOR SELECT TO authenticated USING (true); -- Trigger: auto-fill user_id when member invited by email CREATE OR REPLACE FUNCTION fill_anetaret_user_id() RETURNS TRIGGER AS $$ BEGIN IF NEW.user_id IS NULL THEN SELECT id INTO NEW.user_id FROM auth.users WHERE LOWER(email) = LOWER(NEW.email) LIMIT 1; END IF; RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; DROP TRIGGER IF EXISTS anetaret_fill_user_id ON anetaret; CREATE TRIGGER anetaret_fill_user_id BEFORE INSERT ON anetaret FOR EACH ROW EXECUTE FUNCTION fill_anetaret_user_id(); -- FK for profiles join ALTER TABLE anetaret ADD CONSTRAINT anetaret_user_id_fkey FOREIGN KEY (user_id) REFERENCES auth.users(id);
